# <a name="set-up-fraud-alerts"></a><span data-ttu-id="67ee0-104">Ställ in bedrägerivarningar</span><span class="sxs-lookup"><span data-stu-id="67ee0-104">Set up fraud alerts</span></span>

[!include[banner](includes/banner.md)]

<span data-ttu-id="67ee0-105">Det här avsnittet beskriver hur du ställer in kriterier och regler för att spärra misstänkt bedrägliga försäljningsorder för vidare undersökning.</span><span class="sxs-lookup"><span data-stu-id="67ee0-105">This topic explains how to set up criteria and rules to place potentially fraudulent sales orders on hold for further review.</span></span> <span data-ttu-id="67ee0-106">Funktionen för bedrägerigranskning används för att avgöra informationens giltighet i en försäljningsorder.</span><span class="sxs-lookup"><span data-stu-id="67ee0-106">Fraud review functionality is used to determine the validity of information in a sales order.</span></span> <span data-ttu-id="67ee0-107">Om informationen i försäljningsordern verkar vara tveksam utifrån organisationens regler och kriterier för bedrägeri, ordern kan spärras för vidare undersökning av en administratör.</span><span class="sxs-lookup"><span data-stu-id="67ee0-107">If the information in the sales order appears to be questionable based on an organization’s fraud criteria and rules, the order may be put on hold for further review by an administrator.</span></span>

> [!NOTE]
> <span data-ttu-id="67ee0-108">Funktionen kan endast användas med Retail kundtjänstkanal för bearbetning av försäljningsorder.</span><span class="sxs-lookup"><span data-stu-id="67ee0-108">This feature can only be used with Retail Call Center channel sales order processing.</span></span> 

<span data-ttu-id="67ee0-109">När kundtjänstkanalen definieras måste **aktivera slutförande av order** anges till **Ja**.</span><span class="sxs-lookup"><span data-stu-id="67ee0-109">When the Call Center channel is defined, **Enable Order Completion** must be set to **Yes**.</span></span> <span data-ttu-id="67ee0-110">När slutförande av order är aktiverad kan användare visa ordersammanfattning och klicka på **skicka** för att slutföra ordern.</span><span class="sxs-lookup"><span data-stu-id="67ee0-110">When order completion is enabled, users can view the order recap and click **Submit** to finalize the order.</span></span> <span data-ttu-id="67ee0-111">Användarna får även alternativ för att manuellt spärra försäljningsordern för bedrägerigranskning.</span><span class="sxs-lookup"><span data-stu-id="67ee0-111">Users will also be provided options to manually place the sales order on fraud review hold.</span></span> <span data-ttu-id="67ee0-112">Försäljningsorder som matas in av en kundtjänstanvändare bearbetas via bedrägerikontrollregler och kriterier under sändningsprocessen.</span><span class="sxs-lookup"><span data-stu-id="67ee0-112">Sales orders keyed in by a Call Center user are processed through fraud check rules and criteria during the submit process.</span></span>

<span data-ttu-id="67ee0-113">Det finns två typer av bedrägerikriterier som systemet ska referera till för att kontrollera om ordern ska spärras för bedrägerigranskning:</span><span class="sxs-lookup"><span data-stu-id="67ee0-113">There are two types of fraud criteria that the system will reference to check if the order should be held for fraud review:</span></span>

-   <span data-ttu-id="67ee0-114">**Statistikregler** använd ett specifikt värde, till exempel ett telefonnummer som har blivit svartlistat eller en e-postadress har flaggats som använt för tidigare bedrägliga transaktioner.</span><span class="sxs-lookup"><span data-stu-id="67ee0-114">**Static rules** use a specific value, such as a phone number that has been blacklisted or an email address that has been flagged as being known as used for past fraudulent transactions.</span></span> <span data-ttu-id="67ee0-115">På sidan **Statiska bedrägeridata**, kan bedrägeriinformation läggas till manuellt eller genom dataimport med poäng kopplade till den bedrägliga informationen.</span><span class="sxs-lookup"><span data-stu-id="67ee0-115">On the **Static Fraud Data** page, fraud information can be added manually or through data import, with scores attached to the fraudulent information.</span></span> <span data-ttu-id="67ee0-116">Alla försäljningsorder som angetts ska jämföras med statiska data om kontroll av bedrägeri är aktiverat.</span><span class="sxs-lookup"><span data-stu-id="67ee0-116">If fraud checking is turned on, every sales order entered will be compared to the static data.</span></span> <span data-ttu-id="67ee0-117">Om det finns data i kundens faktureringsadress eller leveransadress eller data i leveransadressen som angetts på försäljningsraden ska resultatet för alla unika träffar summeras.</span><span class="sxs-lookup"><span data-stu-id="67ee0-117">If the data is found in either the customer’s billing or delivery address, or if the data is found in the delivery address on the sales line, the scores of all unique matches will be summed.</span></span>  
-   <span data-ttu-id="67ee0-118">**Dynamiska regler** består av variabler och villkoren för dessa variabler.</span><span class="sxs-lookup"><span data-stu-id="67ee0-118">**Dynamic rules** are composed from variables and conditions defined for those variables.</span></span> <span data-ttu-id="67ee0-119">Andra kriterier som inte har definierats i de statiska reglerna kan kontrolleras med dynamiska regler.</span><span class="sxs-lookup"><span data-stu-id="67ee0-119">With dynamic rules, other criteria not defined in the static rules can be checked.</span></span> <span data-ttu-id="67ee0-120">Mer komplexa ”OCH/ELLER”-utdrag kan användas för att beakta flera villkor för att avgöra om det finns en positiv matchning mot regelvillkoren och försäljningsordern som har skickats.</span><span class="sxs-lookup"><span data-stu-id="67ee0-120">More complex “AND/OR” statements can be used to consider multiple conditions to determine if there is a positive match against the rule criteria and the sales order being submitted.</span></span> <span data-ttu-id="67ee0-121">Till exempel om en användare vill hålla för granskning alla order för kunder som är kopplade till ett visst kundgruppvärde och som har beställt en specifik produkt, kan villkoren för att validera kunden och validera produkterna definieras på sidan **Regler** med ett ”OCH”-villkor.</span><span class="sxs-lookup"><span data-stu-id="67ee0-121">For example, if a user wants to hold for fraud review all orders for customers that are tied to a specific customer group value and that ordered a specific product, the conditions to validate customer and validate products would be defined on the **Rules** page with an “AND” condition.</span></span> <span data-ttu-id="67ee0-122">Ordern skulle endast spärras för bedrägeri om båda villkoren stämmer och om poängvärdet som tilldelats regeln sätter orderns totala bedrägeripoäng över minsta bedrägeripoängen enligt kundtjänsts parametrar.</span><span class="sxs-lookup"><span data-stu-id="67ee0-122">The order would fall into fraud hold only if both conditions were true and if the score value assigned to the rule put the order’s total fraud score over the minimum fraud score as defined in Call Center parameters.</span></span>

<span data-ttu-id="67ee0-123">En kundtjänstanvändare användaren kan manuellt placera en order i kön för spärrad för bedrägeri.</span><span class="sxs-lookup"><span data-stu-id="67ee0-123">A call center user may also manually place an order into the fraud hold queue.</span></span> <span data-ttu-id="67ee0-124">Om användaren som anger ordern anser att kunden som gör ordern är misstänkt och vill att någon ska granska orderns giltighet ytterligare innan den behandlas, kan den användare som anger ordern välja alternativet **Manuell spärr för bedrägeri** från listrutan **Spärrar** på sidan **Sammanfattning av försäljningsorder** (visas efter att orderfunktionen **komplett** kallas).</span><span class="sxs-lookup"><span data-stu-id="67ee0-124">If the user entering the order believes the customer placing the order may be suspicious and wants someone else to further review the validity of the order before it is processed, the user entering the order can choose the **Manual Fraud Hold** option from the **Holds** dropdown menu on the **Sales Order Summary** page (this appears after the **Complete** order function is called).</span></span> <span data-ttu-id="67ee0-125">Användaren uppmanas att ange en notering som ytterligare förklarar varför de anser att ordern kan vara bedräglig så att granskaren har mer sammanhang.</span><span class="sxs-lookup"><span data-stu-id="67ee0-125">The user will be prompted to enter a note to further explain why they feel the order may be fraudulent so that the reviewer has more context.</span></span>

<span data-ttu-id="67ee0-126">Alla ordrar som hålls under manuell bedrägerispärr eller genom systematiska beräkningar av bedrägeriresultat på sidan **orderspärrar** kan granskas och sedan antingen annulleras eller frisläppas för bearbetning.</span><span class="sxs-lookup"><span data-stu-id="67ee0-126">All orders held through manual fraud hold or by systematic calculation of fraud scores will appear on the **Order Holds** page, where the order can be reviewed and then either canceled or released to processing.</span></span>

> <span data-ttu-id="67ee0-127">Användning av flera regler eller alltför komplexa regler leder till dålig systemprestanda när försäljningsorder skickas.</span><span class="sxs-lookup"><span regel utvärderas under funktionen skicka vid kundtjänst försäljningsorderregistrering.</span><span class="sxs-lookup"><span data-stu-id="67ee0-129">It’s important to remember that every rule is evaluated during the submit function of call center sales order entry.</span></span> <span data-ttu-id="67ee0-130">Reglerna utvärderas mot försäljningsorderrubriken och alla orderrader.</span><span class="sxs-lookup"><span data-stu-id="67ee0-130">The rules are evaluated against the sales order header and all order lines.</span></span> <span data-ttu-id="67ee0-131">Det finns flera regler och desto komplexare regelsatser desto längre tid tar de att behandla.</span><span class="sxs-lookup"><span data-stu-id="67ee0-131">The more rules there are and the more complex the rule statements are,
