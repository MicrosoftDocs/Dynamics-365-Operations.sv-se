# <a name="generate-reports-by-adding-content-as-raw-xml"></a><span data-ttu-id="dfcef-103">Skapa rapporter genom att lägga till innehåll som råa XML-data</span><span class="sxs-lookup"><span data-stu-id="dfcef-103">Generate reports by adding content as raw XML</span></span>

[!include[banner](../includes/banner.md)]

<span data-ttu-id="dfcef-104">Du kan använda **RÅA XML**-formatelement för att utforma format för elektronisk rapportering (ER) som skapar utgående dokument i XML-format.</span><span class="sxs-lookup"><span data-stu-id="dfcef-104">You can use the new **RAW XML** format element to design Electronic reporting (ER) formats that generate outgoing documents in XML format.</span></span> <span data-ttu-id="dfcef-105">I vissa fall kanske du vill lägga till obearbetad XML-data i dessa rapporter av en eller flera av följande orsaker:</span><span class="sxs-lookup"><span data-stu-id="dfcef-105">In some cases, you might prefer to add raw XML data to these reports for one or more of the following reasons:</span></span>

- <span data-ttu-id="dfcef-106">Det är bekvämare att använda råa XML-dokument för den ursprungliga designen och löpande underhåll av en rapport eftersom XML-strukturen kan skapas automatiskt genom att köra ett uttryck för körning.</span><span class="sxs-lookup"><span data-stu-id="dfcef-106">It's more convenient to use raw XML for the original design and ongoing maintenance of a report, because the XML structure can be automatically generated by executing a runtime expression.</span></span> <span data-ttu-id="dfcef-107">Därför måste inte flera bindningar fastställas för flera formatelement när du designar.</span><span class="sxs-lookup"><span data-stu-id="dfcef-107">Therefore, multiple bindings don't have to be determined for multiple format elements at design time.</span></span> <span data-ttu-id="dfcef-108">Det är möjligt när du använder datakällor som innehåller information som kan användas för att göra XML-element medan rapporten skapas.</span><span class="sxs-lookup"><span data-stu-id="dfcef-108">It is possible when the data sources that you're using contain information that can be used to make XML elements while the report is generated.</span></span>
- <span data-ttu-id="dfcef-109">Ingen annan metod kan användas för att fylla rapporten med XML-innehåll som tidigare har tagits emot och lagrats i systemet.</span><span class="sxs-lookup"><span data-stu-id="dfcef-109">No other method can be used to fill the report with XML content that was previously received and stored in the system.</span></span> <span data-ttu-id="dfcef-110">Till exempel kanske XML-svaret som har skapats innehåller en XML-begäran som har skickats tidigare.</span><span class="sxs-lookup"><span data-stu-id="dfcef-110">For example, the XML response that is generated might have to contain the content of an XML request that was sent earlier.</span></span>
- <span data-ttu-id="dfcef-111">Ingen annan metod kan användas för att infoga tecken i det skapade dokumentet utifrån deras numeriska koder.</span><span class="sxs-lookup"><span data-stu-id="dfcef-111">No other method can be used to insert characters into the generated document based on their numeric codes.</span></span> <span data-ttu-id="dfcef-112">För vissa språk och tecken existerar inte den här typen av koder.</span><span class="sxs-lookup"><span data-stu-id="dfcef-112">For some languages and characters, codes of this type don't exist.</span></span> <span data-ttu-id="dfcef-113">Exempel inkluderar den grekiska bokstaven rho (ρ) och HTML-koder såsom \&eacute; för ett *e* med akut accent (é).</span><span class="sxs-lookup"><span data-stu-id="dfcef-113">Examples include the Greek letter rho (ρ) and HTML entity codes such as \&eacute; for an *e* that has an acute accent (é).</span></span>

> [!NOTE]
> <span data-ttu-id="dfcef-114">Tänk på att ramverket inte kontrollerar om XML-innehållet som placeras i det skapade dokumentet med hjälp av formatelementet **RÅA XML** är korrekt.</span><span class="sxs-lookup"><span data-stu-id="dfcef-114">Be aware that the framework doesn't control whether the XML content that is placed to the generated document by using the **RAW XML** format element is correct.</span></span>
